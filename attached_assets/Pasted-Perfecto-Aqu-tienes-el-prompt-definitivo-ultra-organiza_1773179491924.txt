## PROMPT MAESTRO: PORTAFOLIO DE CONSULTORÍA TECNOLÓGICA (FRONTEND ONLY)

### 📋 INSTRUCCIONES GENERALES PARA LA IA

Actúa como un **Desarrollador Frontend Senior** especializado en React, Next.js 15, Tailwind CSS, Framer Motion y GSAP. Tu tarea es generar **desde cero** el código completo de un portafolio web para un consultor de estrategia de plataformas y tecnología.

**Stack obligatorio (solo frontend, sin backend ni base de datos):**
- Next.js 15 (App Router) con TypeScript
- Tailwind CSS (para estilos y grid)
- Framer Motion (para animaciones de UI y microinteracciones)
- GSAP + ScrollTrigger (para animaciones avanzadas vinculadas al scroll)
- Lucide React (para iconos)
- Geist font (tipografía principal)

**No incluir:**
- Base de datos (Supabase/Firebase)
- Panel de administración
- API routes complejas
- Autenticación

**Incluir obligatoriamente:**
- Un botón flotante de WhatsApp que dirija al número `+57XXXXXXXXX` (dejar placeholder para que el usuario lo cambie).
- Animaciones de scroll tipo "scroll-triggered" (efecto parallax y rotación de tarjetas vinculada al progreso del scroll), inspiradas en la web `aisummit.com.co`.
- Panel lateral derecho desplegable (fijo).

### 2. IDENTIDAD VISUAL Y DISEÑO GLOBAL

#### 2.1. Paleta de colores (exacta)
- **Fondo principal:** `#030637` (azul marino profundo)
- **Acento primario (verde neón):** `#15f5ba`
- **Acento secundario (magenta):** `#bb00aa`
- **Texto principal:** `#fdfcff` (blanco)

#### 2.3. Grid y layout

- **Tablet (768-1024px):** Grid de 8 columnas, gutter 16px, márgenes 24px.
- **Mobile (<768px):** Grid de 4 columnas, gutter 12px, márgenes 16px. Menú lateral se oculta y se reemplaza por menú hamburguesa (opcional, pero se puede mantener el panel derecho contraído).

### 3. COMPONENTES PRINCIPALES Y SUS ANIMACIONES

#### 3.1. Panel Lateral Derecho Desplegable (`RightSidebar.tsx`)
- **Descripción:** Barra vertical fija a la derecha de la pantalla. Por defecto contraída (solo iconos). Al hacer clic en un icono, se despliega mostrando el texto y, en el caso de "Servicios", un submenú anidado.
- **Iconos necesarios (de Lucide React):**
  - Inicio: `Home`
  - Servicios: `Briefcase` (al desplegar, muestra: Tecnología Restaurantes, Plataformas, Cocinas Ocultas, Economía Creativa)
  - Estudios/Experiencia: `BarChart` (o `TrendingUp`)
  - Insights: `Newspaper`
  - Sobre mí: `User`
  - Contacto: `Mail` (redirige a sección de contacto o abre modal)
- **Animación:** Despliegue suave con Framer Motion (ancho expandido/contraído, opacidad del texto). Duración 0.3s, easing `easeInOut`.

#### 3.2. Botón Flotante de WhatsApp (`WhatsAppButton.tsx`)
- **Descripción:** Botón circular fijo en la esquina inferior derecha (z-index alto). Color de fondo verde neón (`#15f5ba`), icono de WhatsApp. Al hacer clic, abre `https://wa.me/57XXXXXXXXX` (dejar placeholder).
- **Animación:** Leve rebote (scale) cada 5 segundos para llamar la atención. En hover, escala a 1.1.

#### 3.3. Hero Section (`Hero.tsx`)
- **Contenido:**
  - Título: "Arquitectura Estratégica para Plataformas Digitales"
  - Subtítulo: "Consultoría para startups de alta velocidad: Rappi, DiDi, Ghost Kitchens, cocinas ocultas y creadores UGC."
  - CTA doble: "Ver servicios →" (fondo magenta) y "Agendar consultoría" (borde verde neón).
  - Badge: "+15 proyectos implementados | 5 años en LatAm"
- **Fondo:** Canvas con partículas/redes de datos (opcional, se puede simplificar con un gradiente animado si la IA no puede implementar Three.js fácilmente).
- **Animación de entrada:**
  - Partículas: fade-in 1.5s.
  - Título: desde abajo (translateY 40px) con overshoot.
  - Subtítulo y CTAs: entrada con stagger de 0.2s, desde izquierda.

#### 3.4. Sección Servicios - Página Principal (`/servicios/page.tsx`)
- **Grid:** 2 columnas en desktop, 1 en mobile.
- **Tarjetas de servicio (4):** Cada una con icono, título, descripción corta, métrica destacada y botón "Ver más" que enlaza a su sub-página.
  - **Tarjeta 1: Tecnología para Restaurantes**
    - Icono: `Store` o `Utensils`
    - Métrica: "+150% tráfico orgánico"
  - **Tarjeta 2: Estrategia de Plataformas (Rappi/DiDi)**
    - Icono: `Network`
    - Métrica: "+32% GMV"
  - **Tarjeta 3: Cocinas Ocultas (Ghost Kitchens)**
    - Icono: `ChefHat` o `Package`
    - Métrica: "45min tiempo de entrega"
  - **Tarjeta 4: Economía Creativa (UGC)**
    - Icono: `Zap` o `Camera`
    - Métrica: "3.2x ROAS"
- **Animación de entrada (ScrollTrigger):**
  - Las tarjetas aparecen con un **efecto de rotación en Y vinculado al scroll**: cuando la tarjeta entra por la parte inferior del viewport, rotateY = 15deg; cuando está en el centro, rotateY = 0deg. Además, fade de 0 a 1.
  - Stagger de 0.1s entre tarjetas.

#### 3.5. Sub-páginas de Servicio (plantilla reutilizable)
**Rutas:**
- `/servicios/tecnologia-restaurantes`
- `/servicios/plataformas`

#### 3.6. Sección Estudios/Experiencia (`/estudios/page.tsx`)

  - Técnicos: Next.js, React, Estrategia de Plataformas, Growth, SEO, Automatización.
  - Blandos: Liderazgo, Comunicación, Metodologías ágiles.

#### 3.7. Sección Sobre Mí (`/sobre-mi/page.tsx`)
- **Banner de alto impacto** (componente `AboutBanner.tsx`):
  - Fondo: imagen de alta calidad (urbana/tecnológica) con superposición de color `#030637` al 70%.
  - Sobre la imagen, en el centro:

#### 3.8. Sección Insights (`/insights/page.tsx`)

- Cada tarjeta: imagen placeholder, título, extracto, fecha, tiempo de lectura, etiqueta de categoría (Plataformas, Restaurantes, etc.).
- **Animación:** Entrada con fade y ligero scale (0.95 a 1) con ScrollTrigger.

### 4. SISTEMA DE ANIMACIONES AVANZADAS (GSAP + SCROLLTRIGGER)

#### 4.1. Configuración global
- Crear un hook personalizado (ej. `useScrollAnimations`) o inicializar en `ScrollAnimations.tsx` para registrar el plugin ScrollTrigger.

#### 4.2. Efectos específicos

| Elemento | Animación ScrollTrigger |
| :--- | :--- |
| Tarjetas de servicios (grid) | Rotación Y (0° a 15° y viceversa) basada en el progreso del scroll. Usar `scrub: true` para que sea fluido. |
| Títulos de sección | Parallax suave: se mueven a diferente velocidad que el fondo (translateY con factor 0.3). |
| Línea de tiempo | Activación de hitos: al llegar a cada hito, cambiar opacidad y escala, y mostrar descripción. |
| Barras de skills | Ancho de la barra se llena de 0% a su valor final al entrar en viewport. |
| Bloques Bento (si se usan) | Efecto de "olas": entrada desde diferentes direcciones según posición. |

#### 4.3. Microinteracciones (Framer Motion)
- **Hover en tarjetas:** Elevación (translateY -8px) + glow neón (sombra `#15f5ba`).
- **Botones:** Efecto magnético (opcional) o simplemente cambio de escala a 1.05 y color de fondo a magenta.
- **Cursor personalizado:** (opcional, si la IA puede implementarlo) un círculo que sigue al mouse con retraso.

### 8. RESTRICCIONES NEGATIVAS (Lo que NO debe haber)

- **No** usar imágenes de stock genéricas (evitar apretones de manos, oficinas vacías). Usar placeholders o gradientes.
- **No** incluir backend ni llamadas a APIs externas (todo debe ser frontend puro con datos mock).
- **No** usar tipografías serif o manuscritas; solo Geist.
- **No** sombras difusas exageradas; priorizar bordes de neón y glassmorphism.
